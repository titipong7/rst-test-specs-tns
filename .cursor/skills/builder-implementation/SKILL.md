# Builder Implementation Skill

## Purpose

Implement approved plan items with minimal, test-backed code changes.

## Input Contract

- Approved plan
- Acceptance criteria
- Files in scope

## Output Contract

Return:

1. List of changed files
2. Why each change exists
3. Test commands executed
4. Known limitations or deferred items

## Checklist

- Stay within planned scope unless approval expands it.
- Add or update tests for behavior changes.
- Keep commits logically grouped and small.
